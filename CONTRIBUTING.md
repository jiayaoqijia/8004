# Contributing to 8004

Thank you for your interest in contributing! This document provides guidelines and information for contributors.

## Code of Conduct

Be kind, be respectful. We are all here to build useful things for the ERC-8004 ecosystem.

## Ways to Contribute

- **Bug reports**: Found something broken? Open an issue.
- **Feature requests**: Have an idea? Open an issue to discuss it.
- **Code**: Fix bugs, add features, improve documentation.
- **Skills**: Create or improve Claude Code skills for the ecosystem.
- **Documentation**: Improve READMEs, reference docs, and guides.

## Getting Started

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/<your-username>/8004.git
   cd 8004
   ```
3. Add upstream remote:
   ```bash
   git remote add upstream https://github.com/jiayaoqijia/8004.git
   ```
4. Create a branch:
   ```bash
   git checkout -b feat/my-feature
   ```

## Making Changes

### Branching

- Create feature branches from `main`
- Use descriptive branch names: `feat/agent-search`, `fix/chain-resolution`, `docs/api-reference`

### Commits

- Use [Conventional Commits](https://www.conventionalcommits.org/):
  - `feat:` — new feature
  - `fix:` — bug fix
  - `docs:` — documentation
  - `chore:` — maintenance
  - `refactor:` — code restructuring
- Keep commit messages concise and descriptive
- **No AI attribution in commits** — do not add `Co-Authored-By`, `Generated by`, or similar lines

### Skill Development

Skills follow [ClawHub](https://clawhub.ai) standards:

- `SKILL.md` with YAML frontmatter (name, description, version, metadata)
- Supporting reference files in `references/` directory
- Text-based files only (no binaries)
- Lazy-load references via `{baseDir}/references/` pattern

### Keeping Up to Date

```bash
git fetch upstream
git rebase upstream/main
```

## Pull Request Process

1. Ensure your changes work as expected
2. Update relevant documentation
3. Open a PR with a clear title and description:
   - What changed and why
   - How to test
4. Address review feedback

### PR Template

```
## Summary
Brief description of changes.

## Changes
- Item 1
- Item 2

## Test Plan
How to verify the changes work.
```

## AI-Assisted Contributions

This project welcomes AI-assisted contributions. If you use AI tools:

- **You are responsible** for all code you submit, regardless of how it was generated
- Review AI-generated code for correctness, security, and quality
- Ensure the code matches project conventions
- Do not add AI attribution to commits

## Merge Strategy

- Squash merge for feature branches
- Keep `main` clean and linear
- Delete branches after merge

## Questions?

- Open a [GitHub Issue](https://github.com/jiayaoqijia/8004/issues)
- Check existing issues and discussions first
